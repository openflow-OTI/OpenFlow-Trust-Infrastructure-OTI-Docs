# OTI Manager Workspace — replit.md

## Project Overview

This is the **Manager's documentation workspace** for OTI (OpenFlow Trust Infrastructure). It contains no source code. The Manager uses it to write Builder task prompts, review work, and maintain the roadmap.

The actual OTI source code lives in Ahmad's GitHub repos:
- Backend (private): https://github.com/openflow-OTI/OpenFlow-Trust-Infrastructure-OTI-
- Frontend (public): https://github.com/openflow-OTI/OpenFlow-Trust-Infrastructure-O-T-I-Frontend-
- Docs (this workspace): https://github.com/openflow-OTI/OpenFlow-Trust-Infrastructure-OTI-Docs

All docs live in `extracted_docs/docs/`. The docs.zip at root is the version Builders extract from the GitHub repos.

## User Preferences

- **No new files for design briefs or prompts.** All design work, task prompts, and status updates go directly into the existing doc files (`MANAGER_HANDOVER.md`, `TASKS.md`, `BACKEND_TASKS.md`, `FRONTEND_TASKS.md`). Do not create separate design brief files or pending prompt files — update the existing docs in-place.
- Every reply to Ahmad must be in a copy box so he can paste it to Builders.
- One task at a time per Builder — hard rule, never break it.
- Fixes never get task numbers — they go in `FIXES.md` only.
- Manager never touches code or pushes to GitHub.
