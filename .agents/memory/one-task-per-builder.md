## Rule
Never give a Frontend Builder or Backend Builder more than one active task simultaneously. Queue the next task only after the current one is confirmed done AND verified live by the Manager.

**Why:** Ahmad manages Builders on his phone, pasting prompts manually. Two active tasks creates confusion — Builders ask questions about the wrong task, split focus, and Ahmad loses track of what's in progress. Ahmad stated this explicitly and said he will not repeat it.

**How to apply:**
- When a Builder finishes a task and a new one is ready, write the new task prompt only AFTER Ahmad confirms the previous task is verified done.
- If a bug fix comes in while a Builder is mid-task, send only the bug fix prompt — do not include the next queued task in the same message.
- If the Frontend Builder and Backend Builder are both active, each can have their own single task — but never two tasks for one Builder at the same time.
- When in doubt, hold the next task prompt until Ahmad explicitly asks for it.
